Python Exceptions



Section 1.1: Python Exceptions Overview

In Python, there're 2 types of Error mostly used : Syntax errors and Exceptions

Syntax errors occur during invalid syntax, in most cases compiler will underline them.

Exceptions occur during execution time, where you might have logical error.

Section 2.1: Syntax Errors

Syntax errors are perhaps the most common kind of error you get while learning Python.
They occur during invalid syntax and in most cases your compiler will underline them.

print   "Hello World" #    Ignoring parentheses intentionally
#   Output:
>>>  SyntaxError: Missing parentheses in call to 'print'. Did you mean print("Hello World")?

Similarly, wrong indentation might cause to an error:

while   True :
    print ( "Hello World" ) #    Wrong indenting intentionally
#   Output:
>>>  IndentationError: unexpected indent

Section 2.2: Python Exceptions

Python Exceptions consist of 3 main blocks:

  • try - tries to execute a set of code.

  • except - what to do in case of Error.

  • finally (optional) - mandatory executes even if error occurs.

#   simple usage of "try-except-finally" block
try :
    name  =  str ( input ( " Enter your name: " ) )
except   ValueError :
    print ( " Only string allowed " )
finally : #   regardless of try/except, I'll be shown
    print ( " I'll be shown no matter happens! " )

  • Catching Multiple Errors can be done by simply adding commas :

try :
     . . .
except (  ValueError ,  TypeError ,  NameError ,  OSError ) :
     . . .

  • as keyword for error clarification :

try :
    name  =  str ( input ( " Enter your name: " ) )
except   ValueError   as   error  :
    print (  " Error occurred by : "  ,  error  )

  • Without knowing Error Name - we can declare except block, even without knowing its name :

try :
    name  =  str ( input ( " Enter your name: " ) )
except   A :
    print ( " Error occurred! " )


Section 2.3: Finally Block

finally - optional block, meaning you can either declare it or not.

finally - mandatory executes even if error occurs.

try :
    name  =  str ( input ( " Enter your name: " ) )
except   ValueError :
    print ( " Only string allowed " )
finally : #   regardless of try/except, I'll be shown
    print ( " I'll be shown no matter happens! " )

try :
    . . .
except   A :
    . . .
finally : #   regardless of try/except, I'll be shown
    print ( " Good Bye! " )



Section 3.1: Raise Statement

raise - keyword is used to raise an exception when specified condition becomes True.

raise    ExceptionName

try :
    if    x < 0 :
        raise    Exception   ( " Less than zero " )
except :
    . . .

def    Division ( a , b ) :
    if    b == 0 :
        raise   " Division by zero not allowed! "
    else   :
        return a/b
