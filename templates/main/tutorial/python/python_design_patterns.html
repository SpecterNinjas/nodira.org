Python Design Patterns



Section 1.1: Design Patterns Overview

Design Pattern is a general and reusable solution to a commonly occurring problem in a sphere of IT.

In this section, we'll more about Design Patterns on a deep level.

Section 1.2: Design Pattern Development Life Cycle

The Development Life Cycle of a design pattern begins with object creation in a manner suitable to the situation.

Development Life Cycle :

  • Independent System Mainstream

  • Evaluate common modeling tasks and key resources in reference implementations

  • Using the identified resources as key components of the design pattern

  • Include design patterns in plug-in projects, associate the plug-in project to a feature project

  • Test design patterns

Section 2.1: Creational Design Patterns

Creational design patterns are design patterns that deal with object creation mechanisms, trying to create objects in a manner suitable to the situation.

They're composed of two dominant ideas. One is encapsulating knowledge about which concrete classes the system uses. Another is hiding how instances of these concrete classes are created and combined.

Applying Creational Design Patterns when :

  • Independent System Mainstream

  • A set of related objects is designed to be used together

  • Hiding the implementations of a class

  • Diverse representation of independent complex objects

  • Instance should be extensible without being modified

Section 2.2: Structural Design Patterns

Structural design patterns are used to ease the design by identifying a simple way to realize relationships among entities.

They're consisted of a number of design patterns :

  • Adapter pattern :  Adapts one interface for a class into one that a client expects

  • Aggregate pattern :  Applying aggregation of children

  • Bridge pattern :  Abstraction from its implementation so that the two can vary independently

  • Composite pattern :  Tree structure of objects where every object has the same interface

  • Proxy pattern :  Class functioning as an interface to another thing

  • Flyweight pattern :  A large quantity of objects share a common properties object to save space

  • Extensibility pattern :  Hiding complex code behind a simple interface

  • Facade pattern :  Creating simplified interface of an existing interface to ease usage for common tasks

Section 2.3: Behavioral Design Patterns

Behavioral Pattern is a way of identifying common communication patterns among objects and realize these patterns

This pattern is concerned with interaction among objects and responsibility of objects.

They're consisted of a number of design patterns :

  • Blackboard pattern :  provides a computational framework for the design and implementation of systems that integrate large and diverse specialized modules

  • Command pattern :  Command objects encapsulate an action and its parameters

  • Interpreter pattern :   Implement a specialized computer language to rapidly solve a specific set of problems

  • Mediator pattern :  Provides a unified interface to a set of interfaces in a subsystem

  • Memento pattern :   Provides the ability to restore an object to its previous state

  • Observer pattern :  Weak reference pattern

  • Protocol pattern :  Communications are handled by multiple layers, which form an encapsulation hierarchy

  • Strategy pattern :  Algorithms can be selected on the fly, using composition

Section 3.1: Conclusion

Python is a powerful language and it gives you all the power you need to produce flexible and reusable code. Design Patterns are really useful for creating reusable across multiple projects, while providing great reliability and dependency.

Design Patterns can be easily created and executed using Python. Happy coding :)
