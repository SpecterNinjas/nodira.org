Python Global / Local / Nonlocal



Section 1.1: Python Variable Categories

In this chapter, we'll cover python variables.
Basically, python variables are divided into 3 categories: Global, Local and Nonlocal

Section 2.1: Python - Global Variable

  • In this example, a is considered as a global variable, since it was declared outside the function.

a = "Hello World - Python!"     #    global variable
def abc( ):
    print(a)
abc( )      #    Output:    Hello World - Python!

  • In this example, a is considered as a local variable, since it was declared inside the function.

def abc( ):
    a = " I'm local variable - Python! "     #    local variable
    print(a)
abc( )      #    Output:    I'm local variable - Python!

  • Assigning both global and local variable with the same name a.

a = " Global variable - Python! "     #    global variable

def abc( ):
    a = " Local variable - Python! "     #    local variable
    print(a)

abc( )      #    Output:    Local variable - Python!
print(a)     #    Output:    Global variable - Python!

Section 2.2: Python - Global Keyword

  • global keyword is used to inform that any change will occur at that specified global variable.

a = " Hello! "     #    global variable

def abc( ):
    global a
    a = " Bye! "     #    local variable overrides global variable
    print(a)

abc( )      #    Output:    Bye!
print(a)     #    Output:    Bye!

Section 3.1: Python - Local Variable

  • In this example, a is considered as a global variable, since it was declared outside the function.

def abc( ):
a = " Hello World! "     #    local variable
    print(a)
abc( )      #    Output:    Hello World!

  • Since a is declared only inside the function, you can use it only inside the function block, otherwise NameError raises.

def abc( ):
a = " Hello World! "     #    local variable
    print(a)

print(a)     #    Output:    NameError: name 'a' is not defined

Section 3.2: Global vs Local variable

  • Global variable - accessible inside and outside the function.

  • Local variable - accessible only inside the function.

a = " hi "     #    global variable
def abc( ):
    a = " hello "     #    local variable
    print(a)

abc( )      #    Output:    hello
print(a)     #    Output:    hi

  • globals( ).keys( ) - displays all global variables.

  • locals( ).keys( ) - displays all local variables.

a = " hi "     #    global variable
def abc( ):
    a = " hello "     #    local variable
    print(globals( ).keys( ))     #    Output:    shows a list of all global variables
    print(locals( ).keys( ))     #    Output:    shows a list of all local variables

a = " hi "     #    global variable
def abc( ):
    a = " hello "     #    local variable
    print(globals( ).['a'])     #    Output:    hi
    print(locals( ).['a'])     #    Output:    hello

a = " hi "     #    global variable
def abc( ):
    b = " hello "     #    local variable
    print(a in globals( ).keys( ))     #    Output:    True
    print(b not in locals( ).keys( ))     #    Output:    False

Section 4.1: Python - Nonlocal Variable

  • Nonlocal variables are only used in nested functions, where a variable is neither in the local nor the global scope.

def abc( ):
    a = " hi "     #    local variable
    def ab( ):
         nonlocal a
         a = " hello "     #    Nonlocal variable
         print(a)

    ab( )      #    Output:    hello
    print(a)

abc( )      #    Output:    hello

Section 5.1: Summary

  • global a : meaning that it's referencing to a global variable.

  • nonlocal a : meaning that it's referencing to a nonlocal variable (only used in nested functions).

  • Local variable : if variable is declared inside the function.
