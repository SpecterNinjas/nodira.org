Indentation and Comments



Section 1.1: Python Indentation

The Python language has many similarities to C/C++ and Java.
However, there are some differences between the languages.

Firstly, Python uses whitespace indentation, rather than curly brackets or keywords, to delimit blocks and make your code look pretty, smooth and easy to debug. An increase in indentation comes after certain statements; a decrease in indentation signifies the end of the current block.

Indentation is mainly used under statements like if, for, while , try and class. Meaning, The statements with the same indentation belong to the same group.

In general, many programming languages (C/C++, Java) uses curly braces { } to define its contents. But, python breaks this golden rule and uses indentation instead.

if    mood == 'bad' :
    print  ( ' Go back to sleep! ' )
else :
    print  ( ' Start making money! ' )

  • Another example of indentation in Class:

class   MoodClass :
    def  __init( )__ :#   Every function belonging to a class must be indented
        mood = 'Perfect!'#   Everything belonging to __init__ function must be indented

Using spaces or tabs for indentation?

It is highly recommended to use indentation as 4 spaces but tabs or spaces can be used so long as they are consistent.
Do not mix tabs and spaces in Python as this will cause an error in Python 3.x and Python 2.x

Section 1.2: Indentation Errors in Python

Indentation must be uniform throughout statement.Otherwise, it will raise an IndentationError :

a  =  7
if    a > 5  :
    print ( " Correct spacing " )
else   :
    print ( " Correct spacing " )
  print ( " Wrong spacing " )

Or if the line following a colon is not indented, an IndentationError  will also be raised :

a  =  7
if    a > 5  :
print ( " Wrong spacing " )

If you add indentation where it doesn't belong, an IndentationError  will be raised :

a  =  7
if    a > 5  :
    print ( " Correct spacing " )
        print ( " Wrong spacing " )

Section 2.1: Comments in Python

In general, comments are widely used to make the code readable and reusable.

In big companies, one project will be developed by several software engineers and to make your source code look readable and understandable it's a good practise to use comments.

Comments are ignored by compiler and will not raise any syntax error.

  • Single line comment : # this is single line comment. #(hash) is gonna be used.
  • Multi line comment :

    """ This type of comment spans multiple lines. These are mostly used for documentation and created through triple quotes. """

In summary, to create single line comment use # (hash) and for multi line comment use triple quote """ . . . . """

a  =  7
# This is simple single line comment and it does nothing, except making your code readable.
if    a > 5  :
    print ( " Correct spacing " )
