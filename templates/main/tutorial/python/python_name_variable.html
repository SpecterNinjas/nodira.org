Python __name__ variable



Section 1.1: Python "__name__" Overview

In Python, the __name__ is a special variable that is used to check whether a python file is imported or running directly as a main file.

Section 1.2: Python "__name__" in Practice

  • Let's see how it works in practice:

  • We'll create 2 python files : module1.py and module2.py

#   module2.py
if __name__ == "__main__" :
    print ( "Module2 is not imported, it's a main file"  )
else :
    print ( "Module2 is imported"   )

#   module1.py
import   module2 #   without .py extension
if __name__ == "__main__" :
    print ( "Module1 is not imported, it's a main file"  )
else :
    print ( "Module1 is imported"   )

  • If we run module2.py :

#   module2.py


"Module2 is not imported, it's a main file"

  • Let's now run module1.py :

#   module1.py

"Module2 is imported"
"Module1 is not imported, it's a main file"

Section 1.3: Python "__name__" Explanation

In the example above, when we run module2.py, our if condition becomes True since we did't imported any module.
However, while running our second python file module1.py, we first obtain an output coming from imported module, then our compiler goes below and checks for if/else condition and obtains a result.

In Python, the __name__ is a special variable that is used to check whether a python file is imported or running directly as a main file.
