Python Decorators



Section 1.1: Python Decorators Overview

Python provides super cool feature to design and add new functionality to the existed function, called Decorators.
Basically, when it comes to dynamically change the functionality of a function, method or class using decorators are a good practice.

In this section, we'll learn more about how to create and handle decorators on a deep level.

Section 1.2: Python Decorator Syntax

  • The syntax of decorators:

  • @decorator_function

def   decorator_func ( my_func ) :
    return my_func

@decorator_func #   this decorator does nothing
def   my_func ( ) :
    print ( " This is simple decorated function. "  )

  • @ - special sign that's simply equal to the equation below:

my_func  = decorator_func ( my_func )

Section 2.1: Python Decorator Practice

  • Let's create a decorator that'll tell user about the process of a function:

# Simple usage of decorator

def   greeting ( my_func ) :
    def  starting ( *args, **kwargs ) :

        print  ( "Calculating . . . " )

        program  =  my_func ( *args, **kwargs )

        print  ( "Finished! " )

        return program
    return starting

  • Once we defined our decorator, now it's time to see our upcoming output:

@greeting #   this decorator does nothing
def   mul ( a , b ) :
    print ( " Multiplying 2 numbers: " , a * b   )
mul ( 4 , 5 ) #   running the function

#   Output:
Calculating . . .
Multiplying 2 numbers: 20
Finished!

Section 2.2: Python Decorator Class

  • Till now, we only applied decorator to a function, but what if we apply it to a class.   Let's see what happens:

class   Decor ( my_func ) :
    def __init__ ( self , func )
        self.func  = func

    def __call__ ( *args , **kargs )
        print ( "Before"  )
        program  =  self.func ( *args , **kargs )
        print ( "After"  )
        return program

@Decor
def   my_func ( ) :
    print ( "Middle" ) 
#   Output:
Before
Middle
After
