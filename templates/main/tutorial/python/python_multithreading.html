Python Multithreading



Section 1.1: Python Multithreading Overview

This section focuses on Python Multithreading and how to implement in Python projects.

Multithreading is a feature that allows a a program to run multiple tasks concurrently which is very useful for heavy data processing or tasks which rely on external factors, such as slower storage or internet services.

Let's do some practice and learn about Python Multithreading and how to implement & understand them on a deep level.

Section 3.1: Multithreading Advantages - Why We Need ?

In Python, multithreading is the ability of a program or an operating system process to manage its use by more than one user at a time and to even manage multiple requests by the same user without having to have multiple copies of the programming running in the computer.

  • Thread is lightweight - This is a real problem in highly scalable servers, so they don't use threads as if they were lightweight at all. They treat them as heavyweight resources. They might instead create a limited number of threads in a pool, and let them take work items from a queue.

  • Sharing the same memory - Within the shared memory, each thread gets its own stack. Each thread has its own instruction pointer and registers. Since the memory is shared, it is important to note that there is no memory protection among the threads in a process.

  • Saving execution time - In the scenario of multithreading, program execution can be faster. Multithreading method can't guarantee any significant change, but there's a real chance that it'll slow the processing down, because otherwise there's a cost for the program.

Section 3.2: Multithreading Disadvantages - When We Don't Need ?

In general, when we'r using Multithreading in Python, we have to deal with multiple threads, which means it's not going to be easy. Therefore, all threads of data flow within a function will run altogether making it hard to debug.

  • Difficulty of debugging - It is much harder to replicate an error in a multithreaded program than it's to do so in a single-threaded program. As a result, it is more difficult, in the former case, to identify and verify root causes when errors occur.

  • Difficulty of optimization - Existing code often requires significant re-architecting to take advantage of multithreading. Let's suppose in real world projects a single program can have 1000+ files and trying to understand and make a single change may cause to lots of errors.

  • Difficulty of testing - Later on in our course we have an article which focuses on testing your Python code during development process. But, make sure that testing multithreaded program is not going to be simple. It's super hard and requires complex logic.
