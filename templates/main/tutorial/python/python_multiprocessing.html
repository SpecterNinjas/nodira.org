Python Multiprocessing



Section 1.1: Python Multiprocessing Overview

This section focuses on Python Multiprocessing and how to implement in Python projects.

Multiprocessing is a feature that allows a computer has more than one processing engine combined with an operating system that is capable of dispatching work on them such that a computer is actually executing more than one machine instruction at the same instant in time.

Let's do some practice and learn about Python Multiprocessing and how to implement & understand them on a deep level.

Section 2.1: Python Multiprocessing - Pool

Pool object which offers a convenient means of parallelizing the execution of a function across multiple input values, distributing the input data across processes (data parallelism) :

from    multiprocessing   import   Pool
def    func ( x ) :
    return   x + 2

if   __name__  ==  '__main__' :
    with   Pool ( 5 )   as   p :
        print  ( p.map ( func , [ 1 , 2 , 3 ] ) )

By allowing a computer has more than one processing engine to execute this code, we obtain :

#    Output:   [ 3 , 4 , 5 ]

Section 2.2: Python Multiprocessing - Process

Process object provides activity that is run in a separate process.
Process object provides 2 main functions for a trivial example of multiprocessing :

from    multiprocessing   import   Process
def    func ( name ) :
    return   "Hello " + str ( name )

if   __name__  ==  '__main__' :
    p  =  Process ( target  =  func , args  =  ( "James" , "Selena" , "John" ) )
    p.start (  ) #    starts the process's activity
    p.join (  )#    allows processes to join

Section 2.3: Exchanging objects between processes
Queues

Queue - a kind of abstract data type called as First-In-First-Out (FIFO) data structure meaning that the element that enters the queue first , leaves the queue first while deleting the elements in the queue.

Pipes

Pipes - a technique for passing information from one program process to another. Unlike other forms of interprocess communication (IPC), it is one-way communication only.

Section 3.1: Multiprocessing Conclusion

In Python Multiprocessing is easy to write and understand because every process is on its own. A crash in one process does not affect other processes.

Advantages of Multiprocessing

  • Easy to understand & implement in your Python programs.

  • Process Parallelism can be achieved by the kernel through scheduling all the processes simultaneously.

  • Creating several processes can consume time and even exhaust the system resources.

Disadvantages of Multiprocessing

  • Sometimes it might slower the program speed, because switching between processes makes it much slower comparatively.

  • Hard to maintain, whereas in new scenarios you need write more and more new logical algorithms.

  • In some scenarios, it requires lots of effort to debug the code, especially in big projects.
