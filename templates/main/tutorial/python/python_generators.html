Python Generators



Section 1.1: Generators Overview

The Generator functions allow you to declare a function that behaves like an iterator.

Generators is quite simple and straightforward, requiring less memory space.

Let's do some exercises and learn about Python Generators on a deep level.

Section 1.2: Generator Expressions

The Generator Expressions allow you to declare an expression that behaves like list, tuple, set.

  • The syntax of Generator  Expressions :

( expression    for    x   in    iterator )

expression  =  ( x ** 2   for    x   in    range ( 10 ) )

The Generator Functions are similar to regular functions, except that they have one or more yield statements in their body.

  • The syntax of Generator  Functions :

def    double (  ) :
    for    i   in    range ( 10 ) :
        yield    i ** 2

  • This generator function is equivalent to the previous generator expression, it outputs the same.

Section 1.3: Generator Examples

Let's do some practice and solve simple problems :

sum ( i   for   i   in   range ( 10 )   if   i % 2 == 0 ) #   finding  sum  of  even  numbers  in  range ( 10 )
a  =  [ x * 2   for   x   in  [ 1 , 2 , 3 , 4 , 5 ] ] #   Output :   [ 2 , 4 , 6 , 8 , 10 ]     ( list comprehension )
a  =  ( x * 2   for   x   in  [ 1 , 2 , 3 , 4 , 5 ] ) #   Output :   'generator object at 0x7f3833fdd4a0'     ( generator expression )

Section 2.1: Difference between yield and return

  • yield can be used one or more times to create generator function.

  • Both yield and return will return some value from a function.

  • Return sends a specified value back to its caller whereas yield can produce a sequence of values.

  • Yield is used to iterate over a sequence without storing whole sequence in a memory.

Section 2.2: Iterating over generator objects

Iterating over the objects generated by a generator is allowed only once.

If you've already iterated generator objects, you should define the generator function again.

Alternatively, some people declare them in a list for during usage, so that they can iterate over generator objects more than once.

def    double (  ) :
    for    i   in    range ( 10 ) :
        yield    i ** 2

Iterating over generator objects using next (  ) :

gen  =  double (  )
num1  =  next ( gen ) #   Output :    0
num2  =  next ( gen ) #   Output :    1
num3  =  next ( gen ) #   Output :    2
...
num10  =  next ( gen ) #   Output :    StopIteration

next (  )  function is used to receive a value from any iterator :

In example below, we provided a default value ( None ) in case the iterator is out - ranged. If no default is provided, StopIteration going to be raised.

def   nums (  ) :
    yield  1
    yield  2
    yield  3

generator  =  nums (  )
next ( generator , None ) #   1
next ( generator , None ) #   2
next ( generator , None ) #   3
next ( generator , None ) #   None
next ( generator )       #   StopIteration

Section 2.3: Generator Expression Practice

Generator expression is crated using List Comprehension syntax, but under (  ).

expression  =  ( x + 2   for    x   in    range ( 10 ) )
print ( expression )       #   'generator object at 0x8f39703fdd4a0'

next ( expression )       #   0  +  2  =  2
next ( expression )       #   1  +  2  =  3
next ( expression )       #   2  +  2  =  4
. . .
next ( expression )       #   StopIteration

Applying Mathematical functions along with generator expressions :

#   finding  sum  of   numbers  in  range ( 10 )

sum ( i * 2   for   i   in   range ( 10 )  )
#   Output:   0  *  2  +  1  *  2  +  . . .  + 9  *  2  =  90
