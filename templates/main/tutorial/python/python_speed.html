Python Speed Optimization



Section 1.1: Python Speed Overview

Python Speed has become a controversial topic since it's first release 1991.

Since Python is interpreted language it runs a bit slower than C/C++ and Java.

Hence, Speed Optimization greatly contributed and has become more optimized and well-functioned.

Section 1.2: Improving Performance and Scalability

A group software engineers are so obsessed with speed and optimization. While others don't much care about it and choose Python.

They're surprised to find Python code can run at quite acceptable speeds, and in some cases even faster than what they could get from C/C++ with a similar amount of development time invested.

  • Testing with sets & dictionaries is much faster : O ( 1 )

  • Searching from sets & dictionaries is much slower : O ( n )

  • += / + String Concatenation : O ( n )

  • Built-in functions are much faster than Hand-built ones.

  • Queue Applications are much faster : O ( 1 )

Section 2.1: Measure Execution Time

Execution Time can be easily measured by calling time ( ) function before and after a set of code.

Measuring Execution Time of a simple function :

import   time
def   sum ( n )  :
    start_time  =  time.time (  ) #   start measuring time
    summ  =  0
    for   i   in   range ( 1 , n + 1 ) :
        summ  =  summ  +  i
    end_time  =  time.time (  ) #   stop measuring time
    return  summ , end_time  -  start_time

  • Let's now measure execution time :

n  =  10
print ( f" The sum : {n} and measured execution time : {sum ( n )} " )
#    The sum : 10 and measured execution time : ( 55 , 4.76837158203125e-06 )

Section 2.2: Timeit function

Another method for measuring Execution Time can be measured by timeit ( ) function.

Measuring Execution Time of a simple function :

import   timeit
timeit.timeit ( '"-".join ( str ( n )  for   n   in   range ( 100 ) ) ' , number  =  10000 )
#   0.48556045815348625

import   timeit
timeit.timeit ( '"-".join ( [ str ( n )  for   n   in   range ( 100 ) ]  ) ' , number  =  10000 )

import   timeit

Section 2.3: Asymptotic Notations

In general, 3 asymptotic notations are mostly used to represent time complexity of algorithms :

  • Big O Notation : The Big O notation defines an upper bound of an algorithm, it bounds a function only from above. It takes linear time in best case and quadratic time in worst case.

  • Big Omega Notation : The Big Omega notation defines a lower bound of an algorithm, it bounds a function only from below. The best case performance of an algorithm is generally not useful, The Omega notation is the least used notation among all three.

  • Big Theta Notation : The Big Theta notation bounds a functions from above and below, so it defines exact asymptotic behavior. A simple way to get Theta notation of an expression is to drop low order terms and ignore leading constants.
