Python Polymorphism Methods



Section 1.1: Python Polymorphism Overview

This section focuses on Polymorphism method and how to implement in Python projects.

Polymorphism is a feature that allows to define methods in the child class with the same name as defined in their parent class.

Let's do some practice and learn about polymorphism on a deep level.

Section 2.1: Python Polymorphism Practice - Part 1

Let's create a parent class Figure.
Then child classes that inherits from parent class and have method with the same name as defined in Figure.
We add a method area that finds area of each figure:

#   Defining Parent class
class   Figure :
    width , height  =  0
#   Defining Area method
    def   area ( self ) :
        print ( " Parent Class Area " )

class   Triangle ( Figure ) :#     inheriting Parent class
    def   __init__ ( self , width , height ) :
        self.width  =  width
        self.height  =  height

    def   area ( self ) :
        print ( " Triangle Area: ", self.width * self.height * 0.5 )#     method has been overridden

class   Rectangle ( Figure ) :#     inheriting Parent class
    def   __init__ ( self , width , height ) :
        self.width  =  width
        self.height  =  height

    def   area ( self ) :
        print ( " Rectangle Area: ", self.width * self.height )#     method has been overridden

triangle  =  Triangle ( 5 , 6 )
rectangle  =  Rectangle ( 5 , 6 )

rectangle.area (  )#      Triangle Area:  15
triangle.area (  )#      Rectangle Area:  30

Section 2.2: Python Polymorphism Practice - Part 2

In order to understand Polymorphism on a deep level, let's do one more exercise:

#   Defining Parent class
class   Factory :
    revenue , expense  =  0
#   Defining profit method
    def   profit ( self ) :
        print ( " Parent Class Net Profit Info " )

class   Firm ( Factory ) :#     inheriting Parent class
    def   __init__ ( self , revenue , expense ) :
        self.revenue  =  revenue
        self.expense  =  expense

    def   profit ( self ) :
        print ( " Firm Net Profit: ", self.revenue - self.expense )#     method has been overridden

class   Branch ( Factory ) :#     inheriting Parent class
    def   __init__ ( self , revenue , expense ) :
        self.revenue  =  revenue
        self.expense  =  expense

    def   profit ( self ) :
        print ( " Branch Net Profit: ", self.revenue - self.expense - 2000 )#     method has been overridden

firm  =  Firm ( 50000 , 20000 )
branch  =  Branch ( 23000 , 7000 )

firm.profit (  )#      Firm Net Profit:  30000
branch.profit (  )#      Branch Net Profit:  14000

Section 2.3: Polymorphism Conclusion

Polymorphism is the ability to perform an action on an object regardless of its type. This is generally implemented by creating a base class and having two or more subclasses that all implement methods with the same signature.

Advantages of Polymorphism

  • It supports DRY ( Don't Repeat Yourself ) method making your code ( classes, methods ) reusable.

  • Reusable variable name can be applied to store different data type variables.

  • Method overriding works together with inheritance to enable reusability of existing classes without the need for re-compilation.

Disadvantages of Polymorphism

  • Requires complex logical thinking to implement polymorphism in Multiple Classes.

  • Hard to debug, whereas in a team of 25+ software engineers, trying to understand one's logic can be quite difficult and requires lots of effort.
