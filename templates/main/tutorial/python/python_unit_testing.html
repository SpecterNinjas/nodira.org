Python Unit Testing



Section 1.1: Python Unit Testing Overview

This section focuses on Python Unit Testing and how to implement in Python projects.

Unit Test is a feature that exercises a single behavior of a software module. That module is usually a class, and the behavior is usually a public method of the class.

Unit Tests are low level tests, but they force a small piece of application to perform some action and they check the final result. This action is very simple and the results are independent of other pieces of application. Failure in a unit test shows without a doubt which part of the code doesnâ€™t work.

Let's do some practice and learn about Python Unit Tests and how to implement & understand them on a deep level.

Section 1.2: Unit Testing Concepts

  • Test Fixture  -  is something used to consistently test some item, device, or piece of software. While in-line setup is the simplest test fixture to create, it leads to duplication when multiple tests require the same initial data.

  • Test Case  -  is a specification of the inputs, execution conditions, testing procedure, and expected results that define a single test to be executed to achieve a particular software testing objective, such as to exercise a particular program path or to verify compliance with a specific requirement.

  • Test Suite  -  is a collection of test cases that are intended to be used to test a software program to show that it has some specified set of behaviours.

  • Test Runner  -  is a tool that picks up a source code directory that contains unit tests.

Section 2.1: Unit Test Practice - Part 1

We have covered enough theory, let's start with a simple practice :

import   unittest

class   TestUppercase ( unittest.TestCase ) :

    def   test_upper ( self ) : #     method name must start with 'test'
        self.assertEqual ( 'python'.upper (  ) , 'PYTHON' )

    def   test_isupper ( self ) :#     method name must start with 'test'
        self.assertTrue ( 'PYTHON'.isupper (  ) )
        self.assertFalse ( 'Python'.islower (  ) )

if   __name__ == '__main__' :
    unittest.main (  )

Once we run our Unit Test we should get output like this :

#     Output :
test_isupper ( __main__.TestUppercase ) . . . ok
test_upper ( __main__.TestUppercase ) . . . ok

-----------------------------------------

OK

Section 2.2: Unit Test Practice - Part 2

Let's create our own Unit Test for DivisionByZero Error :

import   unittest

def   division ( a , b ) : #     creation our function
    return   a / b

class   TestDivision ( unittest.TestCase ) :

    def   test_division ( self ) :#     method name must start with 'test'
        self.assertRaises ( ZeroDivisionError , division , 1 , 0 )#     ZeroDivisionError occurs if divisor is 0.

if   __name__ == '__main__' :
    unittest.main (  )

Section 2.3: Unit Test Practice - Part 3

Now, we are going to implement 3rd example on Unit Test :

import   unittest

class   TestPractice ( unittest.TestCase ) :

    def   test_equality ( self ) :#     method name must start with 'test'
        self.assertEqual ( 1 + 2 , 3)#     Defining Equality Test.

    def   test_not_equality ( self ) :#     method name must start with 'test'
        self.assertNotEqual ( 1 + 2 , 4)#     Defining Not Equality Test.

    def   test_for_True ( self ) :#     method name must start with 'test'
        self.assertTrue ( 1 + 2  ==  3 , "This text appears in False condition")#     Defining Assert True Test.

    def   test_for_in ( self ) :#     method name must start with 'test'
        self.assertIn ( 1   in   [ 1 , 2 , 3 ] )#     Defining In Test.

    def   test_for_Not_in ( self ) :#     method name must start with 'test'
        self.assertNotIn ( 1   in   [ 1 , 2 , 3 ] )#     Defining Not In Test.

    def   test_almost_equal ( self ) :#     method name must start with 'test'
        self.assertAlmostEqual ( 1 / 3 , 0.3 ) #     Defining Almost Equal Test.

    def   test_not_almost_equal ( self ) :#     method name must start with 'test'
        self.assertNotAlmostEqual ( 5 / 3 , 2 ) #     Defining Not Almost Equal Test.

if   __name__ == '__main__' :
    unittest.main (  )

Section 3.1: Python Unit Test Conclusion

Python Unit Test supports test automation, sharing of setup and shutdown code for tests, aggregation of tests into collections, and independence of the tests from the reporting framework.

Advantages of Python Unit Test

  • Unit Tests will help to reduce the cost of bugs.

  • Main feature of a program to form complete version of documentation.

  • It forces a small piece of application to perform some action and they check the final result.
