Python Debugging



Section 1.1: Debugging Overview

This section focuses on Debugging and how to debug & trace errors in Python.

Python provides interactive pdb module to debug and inspect source code for Python programs.

Let's do some practice and learn about debugging modules on a deep level.

Section 2.1: The Python Debugger

Python debugger pdb.py module can be evoked through :

pip   install   pdb#      installing  python  debugger

python  -m  pdb  main.py

Once we executed this source code, pdb will automatically so called post-mortem debugging

pdb.set_trace (  ) is used to break into the debugger from a running program.

import    pdb #      import debugger

def   divide ( a , b ) :
    pdb.set_trace (  )#      set breakpoint
    return   a / b

print ( divide ( 5 , 6 ) )

Alternatively to pdb.set_trace (  ), you can use breakpoint (  ) built-in function to set breakpoint.

Let's see what we got as a result when we run our python file :

python   main.py
>>> ~/code/main.py(5)divide (  )
-> return a / b
( Pdb )

Section 2.3: Python Remote Debugger

Python Remote Debugger - rpdb ( remote debugger based on pdb module )

rpdb is a wrapper around pdb that re-routes stdin & stdout to a socket handler. By default, debugger on port 4444 :

pip   install   rpdb#      installing  remote  debugger

import    rpdb;   rpdb.set_trace (  ) #      setting  up  remote  debugger

rpdb port number can be changed through:

import    rpdb;   rpdb.set_trace (  ) #      setting  up  remote  debugger
debugger  =  rpdb.Rpdb ( port  =  9999 ) #      changing  port  number
debugger.set_trace (  )

Let's run this in terminal to connect to remote debugger process.

#      run these command on terminal
>>>   nc   127.0.0.1   9999

Output will be popped out like this:

>>> ~/code/remote_debugger.py(3)module(  )
( Pdb )
