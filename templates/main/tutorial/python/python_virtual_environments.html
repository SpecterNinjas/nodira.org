Python Virtual Environments



Section 1.1: Python Virtual Environments Overview

This section focuses on Python Virtual Environments and how to implement in Python projects.

Basically, Python programs will often use packages and modules that don’t come as part of the standard library. Applications will sometimes need a specific version of a library, because the application may require that a particular bug has been fixed or the application may be written using an obsolete version of the library’s interface.

Let's say an application or specific function may require a new version of module. Therefor, the solution for this problem is to create a virtual environment.

Let's do some practice and learn about Python Virtual Environments and how to implement & understand them on a deep level.

Section 2.1: Creating Virtual Environments

venv  is a module used to create and manage virtual environments.

Let's run this command on your current directory to create a virtual environment :

>>> python3  -m  venv  coding-env

Congratulations! We successively learnt how to create virtual environment called : coding

We have created virtual environment, but it's not enough. We have to activate it.

For Windows Operating System :

>>> coding-env\Scripts\activate.bat

For Linux Operating System :

>>> source coding-env/bin/activate

Section 2.2: PIP - Package Manager

Python provides pip module to install, upgrade and remove packages. We already have a special section for PIP

Let's do some practice and execute simple commands :

  • List command will also show all installed packages, like freeze.

>>>   pip list

#   Output:
behave ( 1.2.4 )
enum34 ( 1.0 )
flask ( 0.10.1 )

  • List Outdated command will show outdated packages along with new versions.

>>>   pip list --outdated

#   Output:
behave ( Current: 1.2.4 , Latest: 1.2.5 )
enum34 ( Current: 1.0 , Latest: 1.1)
flask ( Current: 0.10.1 , Latest: 0.10.2 )

Section 2.3: Upgrade & Uninstall Python Package

  • Upgrade with the latest version of a package called PackageName:

>>>   pip install --upgrade PackageName

  • Upgrade pip itself with the latest version pip :

>>>   pip install --upgrade pip

  • Uninstall of a package called PackageName:

>>>   pip uninstall PackageName

Section 3.1: Checking if running inside a virtual environment

In most cases, especially during debugging you might wander and try to check whether you are running inside a virtual environment.

import   sys
sys.prefix
sys.real_prefix

  • Outside a virtual, environment sys.prefix will point to the system python installation and sys.real_prefix is not deﬁned.

  • Inside a virtual environment, sys.prefix will point to the virtual environment python installation and sys.real_prefix will point to the system python installation.

Section 3.2: Python Virtual Environments - Advantages

One of the main advantages of Virtual Environments is to backup up copies of whole computers and use those if a hard disk crashes. They allow you to make historic backups of whole disks without using much additional disk space. So if a virtual machine crashes you can go back to an earlier version and that's it, easy peasy lemon squeezy!
